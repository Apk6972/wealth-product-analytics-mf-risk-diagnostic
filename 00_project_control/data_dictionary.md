# Data Dictionary — MF Risk Diagnostic Module

Schema reference for every raw input, cache artifact, and processed output in the pipeline. This document defines the contract each pipeline stage satisfies in the V1 implementation; no files are generated by this document itself.

```text
API sources → 01_raw_data/api_cache/ → 01_raw_data/scheme_master/ (config)
           → 02_processed_data/ (calculation outputs) → app / Excel / Power BI
```

---

## 1. Config Layer — `01_raw_data/scheme_master/`

### 1.1 `fund_master.csv`

| Column | Type | Description |
|---|---|---|
| `fund_label` | string | Short display name used as the join key across the whole pipeline |
| `scheme_code` | string/int | MFAPI scheme code |
| `expected_scheme_name` | string | Name used to verify API metadata match |
| `category` | string | e.g. Passive Large Cap, Active Large Cap, Flexi Cap, Hybrid / BAF, Small Cap |
| `sleeve` | string | e.g. Core Equity, Diversified Equity, Hybrid Allocation, Satellite Equity |
| `primary_benchmark` | string | Benchmark label used for primary benchmark-relative analytics |
| `secondary_benchmark` | string | Benchmark label used for secondary comparison |
| `include` | boolean | Whether the fund is active in the current pipeline run |
| `verification_status` | enum | `NEEDS_API_METADATA_VERIFICATION` \| `VERIFIED_VIA_MFAPI_METADATA` \| `FAILED_METADATA_CHECK` |

### 1.2 `benchmark_map.csv`

| Column | Type | Description |
|---|---|---|
| `fund_label` | string | Join key to `fund_master.csv` |
| `primary_benchmark` | string | Benchmark label |
| `secondary_benchmark` | string | Benchmark label |
| `benchmark_methodology` | string | Free-text description of what the benchmark represents / fallback rule |
| `tri_required` | boolean | Whether a true TRI series is required for this mapping |
| `proxy_allowed` | boolean | Whether a disclosed proxy/fallback benchmark is acceptable |
| `proxy_warning` | string | Disclosure text shown when a proxy is used (blank if not applicable) |

### 1.3 `portfolio_weights.csv` (referenced by Attribution module)

| Column | Type | Description |
|---|---|---|
| `fund_label` | string | Join key to `fund_master.csv` |
| `weight` | float | Default portfolio allocation weight (sums to 1.0 across included funds) |

Default weights: UTI Nifty 50 Index 0.25, ICICI Bluechip 0.20, Parag Parikh Flexi Cap 0.25, HDFC Balanced Advantage 0.15, Nippon India Small Cap 0.15.

### 1.4 `stress_scenarios.csv`

| Column | Type | Description |
|---|---|---|
| `scenario` | string | Scenario name (e.g. Broad Equity Correction) |
| `UTI Nifty 50 Index` | float | Per-fund shock (decimal, e.g. -0.15 = -15%) |
| `ICICI Bluechip` | float | Per-fund shock |
| `Parag Parikh Flexi Cap` | float | Per-fund shock |
| `HDFC Balanced Advantage` | float | Per-fund shock |
| `Nippon India Small Cap` | float | Per-fund shock |
| `rationale` | string | Short explanation of the scenario's narrative |

## 2. Raw API Cache Layer — `01_raw_data/api_cache/`

### 2.1 `mutual_funds/` (one cache artifact per `scheme_code`)

Raw MFAPI response payload persisted verbatim (JSON), plus a fetch-timestamp sidecar used to evaluate the 24-hour cache rule.

### 2.2 `benchmarks/` (one cache artifact per `benchmark_label`)

Raw fetched TRI/price-index payload (source-dependent format) plus a fetch-timestamp sidecar.

### 2.3 `metadata/`

Cached scheme metadata responses used for the `expected_scheme_name` verification check, plus verification results.

## 3. Normalized Intermediate Schemas (produced by `api_fetch.py`)

### 3.1 Mutual Fund NAV — normalized per-fund frame

| Column | Type | Description |
|---|---|---|
| `date` | date | Trading date |
| `fund_label` | string | Join key |
| `scheme_code` | string | MFAPI scheme code |
| `scheme_name` | string | Scheme name as returned by MFAPI |
| `nav` | float | Net asset value |
| `source` | string | `MFAPI` |
| `source_quality` | enum | `API_FETCHED_VERIFIED` \| `API_FETCHED_METADATA_WARNING` \| `CACHE_FRESH` \| `CACHE_EXPIRED_USED_AFTER_FETCH_FAILURE` |

### 3.2 Benchmark series — normalized per-benchmark frame

| Column | Type | Description |
|---|---|---|
| `date` | date | Trading date |
| `benchmark_label` | string | e.g. `NIFTY50_TRI`, `HYBRID_65_35` |
| `tri_value` | float | Total-return index level (or price-index level if proxy) |
| `source` | string | e.g. `NSE_TRI_FETCH`, `YFINANCE`, `INTERNAL_SYNTHETIC_BENCHMARK` |
| `source_quality` | enum | e.g. `API_FETCHED_VERIFIED`, `PRICE_INDEX_PROXY_NOT_TRI`, `DISCLOSED_APPROXIMATION` |

## 4. Processed Data Layer — `02_processed_data/`

### 4.1 `nav_daily_clean.csv`

Validated, deduplicated, schema-enforced version of §3.1, filtered to `date >= 2021-01-01`.

Columns: `date, fund_label, scheme_code, scheme_name, nav, source, source_quality`

### 4.2 `benchmark_daily.csv`

Validated, deduplicated, schema-enforced version of §3.2, filtered to `date >= 2021-01-01`.

Columns: `date, benchmark_label, tri_value, source, source_quality`

### 4.3 `returns_daily.csv`

Columns: `date, fund_label, nav, daily_return`

### 4.4 `nav_monthly.csv`

Columns: `month_end_date, fund_label, month_end_nav`

### 4.5 `returns_monthly.csv`

Columns: `month_end_date, fund_label, monthly_return`

### 4.6 `benchmark_monthly.csv`

Columns: `month_end_date, benchmark_label, month_end_tri_value, monthly_return`

### 4.7 `rolling_metrics.csv`

Columns: `date, fund_label, rolling_3m_return, rolling_6m_return, rolling_12m_return, rolling_24m_return, rolling_36m_return, rolling_12m_return_ann, rolling_24m_return_ann, rolling_36m_return_ann, rolling_63d_vol, rolling_126d_vol, rolling_252d_vol, rolling_252d_sharpe`

### 4.8 `benchmark_metrics.csv`

Columns: `fund_label, benchmark_label, excess_return_ann, beta, tracking_error, information_ratio, upside_capture, downside_capture`

### 4.9 `rolling_benchmark_metrics.csv`

Columns: `date, fund_label, benchmark_label, rolling_252d_beta, rolling_252d_tracking_error, rolling_252d_information_ratio`

### 4.10 `metrics_summary.csv`

Columns: `fund_label, cagr, annualized_volatility, downside_deviation, sharpe_ratio, sortino_ratio, max_drawdown, recovery_period_days, best_month, worst_month, positive_month_ratio, daily_var_95, daily_cvar_95, monthly_var_95, monthly_cvar_95, risk_free_rate_used, as_of_date`

### 4.11 `stress_results.csv`

Columns: `scenario_type (HISTORICAL_REPLAY | DETERMINISTIC | CUSTOM), scenario_name, fund_label, stress_return, window_start, window_end, rationale`

### 4.12 `attribution_results.csv`

Columns: `scenario_name, fund_label, fund_weight, fund_stress_return, fund_loss_contribution, stress_loss_share, base_portfolio_value, loss_amount_inr, total_portfolio_stress_return, post_stress_portfolio_value`

### 4.13 `suitability_results.csv`

Columns: `fund_label, client_profile, suitability_role, recommended_action, max_drawdown, volatility, daily_cvar_95, small_cap_exposure_flag, rolling_beta, downside_capture, stress_loss_share, recovery_period_days, finding, interpretation`

### 4.14 `data_quality_report.csv`

Columns: `entity_label (fund_label or benchmark_label), asset_type (FUND | BENCHMARK), source, source_quality, first_date, last_date, observation_count, missing_value_count, duplicate_date_count, suspicious_return_count_gt_10pct, suspicious_return_count_lt_minus_10pct, metadata_verification_status, status (PASS | WARNING | FAIL)`

## 5. Cross-File Join Keys

| Key | Used to join |
|---|---|
| `fund_label` | `fund_master.csv` ↔ `benchmark_map.csv` ↔ `portfolio_weights.csv` ↔ all `02_processed_data/*.csv` fund-level tables |
| `benchmark_label` | `benchmark_map.csv` (via `primary_benchmark`/`secondary_benchmark`) ↔ `benchmark_daily.csv` ↔ `benchmark_metrics.csv` ↔ `rolling_benchmark_metrics.csv` |
| `date` / `month_end_date` | All time-series tables, used for alignment before any covariance/ratio calculation |
| `scenario_name` | `stress_scenarios.csv` ↔ `stress_results.csv` ↔ `attribution_results.csv` |

## 6. Enumerations Reference

| Field | Allowed values |
|---|---|
| `verification_status` | `NEEDS_API_METADATA_VERIFICATION`, `VERIFIED_VIA_MFAPI_METADATA`, `FAILED_METADATA_CHECK` |
| NAV `source_quality` | `API_FETCHED_VERIFIED`, `API_FETCHED_METADATA_WARNING`, `CACHE_FRESH`, `CACHE_EXPIRED_USED_AFTER_FETCH_FAILURE` |
| Benchmark `source_quality` | `API_FETCHED_VERIFIED`, `PRICE_INDEX_PROXY_NOT_TRI`, `DISCLOSED_APPROXIMATION`, `CACHE_FRESH`, `CACHE_EXPIRED_USED_AFTER_FETCH_FAILURE` |
| `data_quality_report.status` | `PASS`, `WARNING`, `FAIL` |
| `scenario_type` (stress) | `HISTORICAL_REPLAY`, `DETERMINISTIC`, `CUSTOM` |
| `client_profile` | `Conservative`, `Balanced`, `Growth`, `Aggressive` |
| `suitability_role` | `Core`, `Satellite`, `Defensive sleeve`, `Aggressive satellite`, `Watchlist`, `Unsuitable for profile` |
| `recommended_action` | `Retain`, `Cap exposure`, `Stagger allocation`, `Pair with defensive sleeve`, `Review benchmark-relative behaviour`, `Avoid for low drawdown tolerance` |
